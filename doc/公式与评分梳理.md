## 1. `dual-lens-value-screener`（双视角筛选器）

### 1.1 格雷厄姆维度（`domain/graham.py`）

| 名称 | 公式 / 规则 | 说明 |
|------|----------------|------|
| NCAV（简化） | `NCAV = 流动资产合计 − 负债合计` | 经典 net-net 的粗口径；文献对负债定义可有更严处理 |
| 市值 / NCAV | `market_cap / NCAV`（NCAV>0 且市值>0） | 比值越低越偏「烟蒂」倾向；小于 1 时记 `net_net_tendency` |
| 流动比率 | `流动资产合计 / 流动负债合计` | 分母为 0 则无值 |
| 市净率 P/B | `市值 / 股东权益合计` | 权益 ≤0 则无值 |
| 综合分（0–100） | 三项子分按权重加权平均 | 子分：NCAV 比线性映射（默认 ≤0.67 满分、≥2.0 零分）；流动比率 capped 后线性；P/B 线性映射（默认 ≤0.5 满分、≥3.0 零分）。缺 NCAV 可比值时用部分分。权重默认：NCAV 0.5、流动比率 0.2、P/B 0.3，可由 `GrahamScoringParams` / 环境变量覆盖（见 `domain/scoring_params.py`） |

### 1.2 巴菲特维度（质量启发式，`domain/buffett.py`）

| 名称 | 公式 | 说明 |
|------|------|------|
| ROE | `净利润_TTM / 股东权益合计` | 权益 ≤0 无值 |
| 杠杆（负债/权益） | 优先 `有息负债 / 权益`；否则 `负债合计 / 权益`（粗代理，记 `debt_proxy_crude`） | |
| 经营现金流 / 净利润 | `经营现金流_TTM / 净利润_TTM` | \|净利润\| 极小则无值；低于 0.5 记 `weak_cash_conversion` |
| 综合分（0–100） | 子分按权重加权 | ROE：相对 `roe_excellent`（默认 20%）线性到 100；杠杆：在 `debt_low`~`debt_high` 间反向线性；OCF/NI：≥1 为 100，(0,1) 间线性。权重默认 ROE 0.45、杠杆 0.30、OCF/NI 0.25（`BuffettScoringParams`） |

**说明**：本维度为**质量与财务稳健性启发式**，**不包含**将未来现金流折现得到的每股内在价值（DCF）。OpenSpec 变更 `add-dcf-intrinsic-valuation` 拟在详情等场景补充简化 DCF，见下文第 4 节。

### 1.3 双视角综合分（`domain/assessment_coverage.py`）

\[
\text{combined} = w_b \cdot \text{buffett\_score} + w_g \cdot \text{graham\_score}
\]

- 默认 \(w_b = w_g = 0.5\)，由 `CombinedRankingParams`（环境变量 `VALUE_SCREENER_COMBINED_WEIGHT_*`）配置，和须约等于 1。

### 1.4 第三套透镜（`domain/third_lens.py` + `application/attach_third_lens_scores.py`）

| 名称 | 公式 | 说明 |
|------|------|------|
| 营收同比（两年年报） | `(营收_新 − 营收_旧) / 营收_旧` | 旧期营收须为正；取 `fs_income` 中最近两条 1231 年报 |
| 盈利收益率 E/P | `净利润_最近年报 / 市值` | 须均为正；市值来自 `provenance_json.market_cap` |
| 百分位（0–100） | `100 × (below + equal×0.5) / n` | 在行业桶内样本上对单只公司的 yoy 或 E/P 排名 |
| 第三套子分合成 | `w_g × growth\_score + w_v × valuation\_score` | 仅一侧有值时该侧权重视为 1；皆无则第三套分为空。默认 \(w_g=w_v=0.5\)（`ThirdLensSubWeights`） |
| 三元综合分 | `w_b·B + w_g·G + w_t·T`；若第三套为空则对 \(w_b,w_g\) 重归一 | `TripleCompositeParams`，默认各 1/3 |

### 1.5 与「是否低估」的关系

- **格雷厄姆分**：偏「相对资产/清算概念的安全边际与便宜度」代理，**不是** DCF 内在价值。
- **巴菲特分**：偏「生意质量与现金流兑现」，**不是** 市值对折现值的折价率。
- **第三套**：行业内**成长与盈利收益率的相对位置**，用于横向比较。
- 三者组合适合**排序与初筛**；若要「内在价值 vs 市值」的同一量纲，需额外估值模块（规划中 DCF 即为此类补充）。

## 2. `ai-hedge-fund`（多分析师 Agent）

### 2.1 估值分析师聚合（`src/agents/valuation.py`）

四法加权后再与市值比较得到 gap 与信号（DCF 与 Owner Earnings 权重各 35%，EV/EBITDA 20%，剩余收益 10%）。

**Owner earnings 估值**

- 单期：`owner_earnings = net_income + depreciation − capex − Δworking_capital`
- 未来 5 年按固定增长率复利折现（折现率默认 15%）；终值 Gordon：`term = OE_T × (1+g_T) / (r − g_T)`，其中 \(g_T = \min(g, 0.03)\)
- 结果再乘以 `(1 − margin_of_safety)`，默认安全边际 25%

**经典 DCF（FCF）**

- 第 \(t\) 年：\(FCF_t = FCF_0 \times (1+g)^t\)，折现率 \(d\)（默认 10%），预测期默认 5 年
- 终值：\(\displaystyle TV = \frac{FCF_0(1+g)^N(1+g_\infty)}{d - g_\infty}\)，再折现到现值
- \(FCF \le 0\) 时返回 0

**EV/EBITDA 隐含股权价值**

- `EBITDA_now = EV / (EV/EBITDA)`；用历史 TTM 样本的 **EV/EBITDA 中位数** × 当前 EBITDA 得隐含 EV
- `net_debt ≈ EV − market_cap`；股权价值 `max(EV_implied − net_debt, 0)`

**剩余收益模型（简化 EBO）**

- `book_val = market_cap / P/B`；\(RI_0 = NI - r_e \times book\)
- 预测期折现剩余收益 + 终值剩余收益；结果 ×0.8 作 20% 安全边际

**聚合**

- 各法 `gap = (value − market_cap) / market_cap`；按有效权重加权平均 gap，阈值 ±15% 判 bullish/bearish/neutral

### 2.2 其他 Agent 中的估值变体（节选）

- **`warren_buffett.py`**：`calculate_owner_earnings` 基础上 **三阶段增长** DCF（阶段 1/2 各 5 年，终值 Gordon），折现率默认 10%，整体再打 **0.85** 保守系数；输出与 **每股** 相关逻辑见该文件内 `owner_earnings / shares` 分支。
- **`bill_ackman.py`**：5 年 FCF 增长折现 + **终值倍数**（×15）而非永续增长公式。
- **`charlie_munger.py`**：多年 FCF 均值作 normalized FCF，再结合 FCF yield 与倍数启发式。
- 另有 **Graham / Lynch / Fisher / Wood / Druckenmiller / Jhunjhunwala** 等文件内含各自评分与简化估值逻辑，口径与参数各不相同，未在此逐条展开；需要时可按文件名检索 `intrinsic`、`valuation`、`margin_of_safety` 等关键字。

## 4. 常见概念澄清（探索用）

- **DCF（现金流折现）** 是公司财务与估值教材中的通用框架，**并非**某个人专有；巴菲特更常强调 **owner earnings（所有者收益）**、**护城河** 与 **安全边际**，其团队也会用多种估值方式交叉验证。
- **本仓库**：`ai-hedge-fund` 已含多种 DCF/终值/倍数实现；`dual-lens-value-screener` **当前**以双视角 + 第三透镜 **打分为核心**，DCF 仅在 OpenSpec 提案阶段。
