# Tushare Pro：积分、权限与频次说明（整理稿）

> **说明**：本文根据 [Tushare 权限说明（doc_id=108）](https://tushare.pro/document/1?doc_id=108) 及社区公开文档整理，便于本地查阅与排障。**细则以官网最新版为准**，积分档位与接口列表可能变更。

---

## 1. 积分在平台中的含义

- 积分体现**数据权限**，也代表对社区的支持与贡献度。
- Pro 数据由社区生产、开发与维护，服务器与带宽成本较高，因此设置**积分门槛**与**调用频次**。
- 当前积分主要用于**门槛与频次分级**；文档表述为分级门槛，**并不消耗积分**（以官网最新说明为准）。

---

## 2. 两种权限模式

| 模式 | 含义 | 典型例子 |
|------|------|----------|
| **需要积分的接口** | 达到对应积分门槛即可调用（在自身频次/日量限制内） | 各类资产日线、基础数据、财务表等（见各接口「最低分值」表） |
| **需单独开通的接口** | **与积分无关**（或另计），**分别开通、各自独立** | 分钟数据、新闻舆情、公告、部分港美股/实时类等（见下「独立权限」） |

## 3. 表（一）：积分档位与调用额度（非分钟权限范畴）

以下摘录常见档位；**每分钟频次**与**每日上限**以官网「积分频次表」为准。

| 积分数 | 每分钟频次 | 每天总量上限 | 可访问范围（摘要） | 捐助参考（元/年） |
|--------|------------|--------------|--------------------|-------------------|
| **120** | 50 | 8000 次 | 股票**非复权日线**等；其它接口按文档可能无法调取 | 0 |
| **2000 以上** | **200** | **100000 次 / 每个 API** | 达到各接口文档要求的积分即可参考调用 | 200 |
| **5000 以上** | **500** | 常规数据**无上限**（文档表述） | 同上；除分钟数据、特色数据外，频次相对更高 | 500 |
| **10000 以上** | 500 | 常规数据无上限；**特色数据**约 300 次/分钟 | 含盈利预测、筹码相关、券商金股等特色数据权限 | 1000 |
| **15000 以上** | 500 | 特色数据**无总量限制** | 特色数据专属权限 | 1500 |

### 3.1 与本项目排障的关系（经验对齐）

- 若报错为：**「抱歉，您每分钟最多访问该接口 200 次」**，与 **2000 分以上档位的「每分钟 200 次」** 一致，多为**分钟配额**触顶，而非「没有接口权限」。
- **「100000 次/个 API」** 通常理解为：**按 API 名称分别计数**的日上限；全市场逐票多接口抓取时，**往往先撞分钟 200**，再才需要关心单日分接口上限（需以官方细则为准）。

## 4. 表（二）：独立权限接口（与积分表一分离）

以下类型通常需**单独付费/开通**，**不在积分表一覆盖范围内**（或另有规则），且各类权限**独立**：

| 类型 | 内容摘要 | 备注（摘自公开说明） |
|------|----------|----------------------|
| 股票历史/实时分钟 | 1/5/15/30/60 分钟等 | 单独年费档位，频次如每分钟 500 次等 |
| 股票/指数/ETF 实时日线 | 开盘后当日实时等 | 多按月费形式 |
| 期货/期权/申万分钟 | 历史与实时 | 单独开通 |
| 港股日线/分钟/财报/实时 | 日线、分钟、财报等 | 港股财报等或需单独费用或高积分（以官网为准） |
| 美股日线/财报 | 全市场历史等 | 单独开通 |
| 新闻资讯、公告 | 快讯、新闻、公告 PDF 链接等 | 单独开通 |
| 其它 | 可转债价格变动、盘前股本、e 互动、集合竞价、政策法规、研报库等 | 各自独立计价/条件 |

**注意**：分钟、港美股等权限**不以积分表一代替**；需要哪类数据，应对照官网「独立权限」页面逐项确认。

## 6. 调用策略上的提醒（工程视角，非官方承诺）

- **每分钟 200 次**为档位上限时，**并行线程/多进程**会叠加请求速率，容易触发限流。
- **单标的**若连续调用 `balancesheet`、`income`、`cashflow` 等多个接口，会占用**多次**分钟配额。
- 遇到「每分钟最多…次」类错误时，**短间隔重试**可能仍在同一分钟窗口内；需结合**全局限速**或**等到下一分钟**等策略（实现见应用代码，不在本文展开）。
