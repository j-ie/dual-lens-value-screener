## Context

当前系统已具备投资质量单公司计算接口、批量任务接口和结果分页接口，但前端页面偏“操作台”，缺少报告化信息组织。用户目标是形成可复盘的分析页面，既能看单公司结论，也能按批次任务查看全量结果，并且能够看到评分原因与数据证据。

约束条件：
- 需兼容现有接口与页面，不破坏已上线流程。
- 报告页应优先复用现有后端能力，减少新增复杂度。
- 支持中文业务语义，结论与风险提示可直接用于业务沟通。

## Goals / Non-Goals

**Goals:**
- 提供独立“投资质量报告”入口，承载报告化展示。
- 统一支持单公司与批量任务两种分析报告路径。
- 报告内容至少包含：总览结论、模块评分拆解、风险提示、关键数据证据。
- 支持从 run 维度检索和切换报告数据。
- 通过“投资大师”AI 层生成价值投资风格总结，并强制输出仓位、买入触发区间和退出条件。

**Non-Goals:**
- 本阶段不引入新的评分模型或重写领域评分算法。
- 本阶段不实现 PDF/Word 导出能力。
- 本阶段不对历史 run 做离线回填修复。

## Decisions

1. 新增独立报告页，而不是在现有“投资质量”页面持续堆叠功能。  
   - 原因：把“计算触发”与“报告消费”分离，降低页面复杂度，便于后续扩展。  
   - 备选：在原页面内加更多区块；缺点是状态管理和交互路径会迅速复杂化。

2. 报告页采用“双入口、同结构”策略。  
   - 单公司路径：输入代码触发 `/api/v1/investment-quality/single`，展示即时报告。  
   - 批量路径：选择 `run_id` 调用 `/api/v1/investment-quality/runs/{run_id}/results`，展示任务报告。  
   - 原因：统一用户心智，减少学习成本。

3. 报告展示分三层信息结构。  
   - 第一层（结论）：决策标签、总分、低估标记、风险计数。  
   - 第二层（解释）：模块分、原因列表、风险标记详情。  
   - 第三层（证据）：市值、行业、公司名等字段与任务上下文。  
   - 原因：先“看结论”，再“看原因”，最后“看证据”，符合分析工作流。

4. 后端优先复用现有 API，必要时增加轻量聚合字段。  
   - 原因：缩短交付时间，降低回归风险。  
   - 备选：新增完整报告聚合服务；适合后续版本，不适合本次快速落地。

5. 报告总结采用“结构化数据先行、AI 叙事后置”的双层模式。  
   - AI 输入只允许使用结构化字段（分数、风险、估值、数据证据），不直接基于页面自由文本。  
   - 原因：确保可解释、可复盘、可测试，降低幻觉风险。  
   - 备选：仅靠模板前端拼接文案；缺点是表达力不足，难覆盖反方论证与情景建议。

6. 新增“投资大师 skill”作为提示词规范，不把业务规则散落在代码中。  
   - skill 固化价值投资风格：先风险后收益、重安全边际、区分事实与判断。  
   - 输出结构固定包含：结论、估值判断、仓位建议、买入区间、反方观点、退出条件、跟踪清单。

## Risks / Trade-offs

- [风险] 单公司路径数据可能受上游数据源波动影响，导致实时结果延迟或失败。  
  → 缓解：前端明确错误原因并支持重试；保留上次成功结果视图。

- [风险] 报告页信息量大，首次渲染体验可能变慢。  
  → 缓解：按区块懒加载与骨架屏，优先渲染结论层。

- [风险] 批量 run 切换频繁时会造成重复请求。  
  → 缓解：引入前端缓存键（run_id + page + sort）并复用现有后端缓存策略。

- [权衡] 复用现有接口可快速交付，但报告聚合能力有限。  
  → 结论：先满足 MVP，后续按使用反馈再扩展专用报告接口。

- [风险] AI 总结可能给出过强确定性措辞，影响决策纪律。  
  → 缓解：skill 强制使用条件句、置信度标签，并输出“假设被证伪”条件。

- [风险] AI 输出若未绑定证据字段，可能出现无法复盘的观点。  
  → 缓解：要求每段结论至少映射一个结构化字段来源。

## Migration Plan

1. 新增报告页路由与导航入口，保持旧入口不变。  
2. 接入现有单公司与批量结果接口，完成报告 UI 渲染。  
3. 增加 AI 总结接口与“投资大师 skill”配置，完成报告叙事层。  
4. 联调并补充测试用例，确保“单公司/批量 + AI 总结”路径可用。  
5. 发布后观察报错与性能指标，再决定是否追加后端聚合增强。

回滚策略：关闭新 Tab 导航入口并回退前端构建版本，不影响现有 API 与页面。

## Open Questions

- 是否需要在 MVP 阶段提供报告下载（CSV/PDF）？  
- 报告页是否需要与 AI 历史页做深链跳转（同 symbol 同 run）？  
- 批量报告默认排序应以综合分还是投资质量总分为主？  
- 仓位建议输出是否采用固定档位（观察仓/试探仓/标准仓）还是百分比区间？
