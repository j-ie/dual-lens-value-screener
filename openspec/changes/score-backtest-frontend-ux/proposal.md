## Why

当前分数策略回测将进入实施阶段，但如果仅提供后端能力，策略研究与参数迭代仍然依赖命令行和手工拼接，操作成本高、反馈慢。需要同步建设可操作的前端工作台，让研究人员可以低门槛完成回测配置、执行、对比与复盘。

## What Changes

- 新增回测前端工作台能力，支持创建回测任务、查看执行进度、查询结果摘要。
- 新增回测参数表单能力，支持策略模式、调仓频率、持有期、成本、基准与过滤条件配置。
- 新增回测结果展示能力，支持净值曲线、分层收益、IC/RankIC、核心绩效指标可视化。
- 新增回测任务列表与详情能力，支持状态筛选、参数回显、失败原因展示与重跑入口。
- 补充后端接口契约，确保前端可按统一结构读取任务状态与结果数据。

## Capabilities

### New Capabilities
- `backtest-frontend-workbench`: 提供回测任务配置、执行、监控、结果查看与复跑的前端操作闭环。

### Modified Capabilities
- （无）

## Impact

- Affected code:
  - `frontend/src/`：新增回测工作台页面、参数表单、结果图表组件、任务列表与详情交互。
  - `src/value_screener/interfaces/`：新增或扩展回测任务创建/查询/结果接口。
  - `src/value_screener/application/`：补充用于前端展示的任务与结果聚合查询服务。
  - `src/value_screener/infrastructure/`：补充回测任务与结果读取仓储能力。
- API contract:
  - 增加前端可消费的统一响应结构（任务状态、参数快照、指标摘要、曲线与诊断数据）。
- Dependencies:
  - 复用既有回测领域能力与投资质量评分能力，不新增外部前端依赖作为首版前提。
